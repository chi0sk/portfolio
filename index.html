01 / about

who i am

i go by chi0sk and humaninput. most of what i like building is the stuff under everything else: serialization, networking, state sync, persistence, queues, safety checks, and the parts people usually leave until they become a problem.

i am not really interested in flashy code that falls apart later. i like systems that are easy to reason about, easy to extend, and still readable when the codebase gets bigger. that matters more to me than trying to look clever.

i do roblox commissions and i am open to other programming work too. if you need something fast and rough, i am probably not your guy. if you want it built clean and built to hold up, that is where i am useful.

02 / work

modules

001

Loom

A typed binary serializer for roblox with remote wrappers and higher level channels on top. It covers schemas, tracked deltas, delete support, roblox value types, and cleaner remote payloads in general. I made it because i was tired of sending loose tables everywhere and pretending that was a real long term solution.

-- state sync with typed fields local playerState = LoomChannels.state(remotes.PlayerState, { {"health", Loom.u16}, {"stamina", Loom.u16}, {"position", Loom.vec3}, })
roblox module 2026
002

NetReplication

Server-owned state replication built around replicas. Clients subscribe, get batched updates, and recover automatically if they miss something or detect bad data. Delta compression, integrity checks, and refresh logic are all built in. It came from getting sick of rewriting the same sync layer over and over.

-- server updates a value, clients get the change replica:Set({ "coins" }, 500) print(replica:Get({ "coins" }))
roblox module 2025
003

ChronicleStore

A datastore wrapper built around session ownership and stale write protection. It grabs a lease before touching a record, rejects old writes, retries sanely, and keeps the ugly parts of `UpdateAsync` in one place instead of leaking all over the project.

local profile = Store:StartSessionAsync("player_123") profile.Data.coins = profile.Data.coins + 100 profile:Save()
roblox module 2025
004

TaskScheduler

A distributed task queue for roblox servers. You can submit work with retries and priorities, register handlers, and let workers pull from the queue. good for jobs you do not want blocking gameplay code or depending on one server staying alive forever.

scheduler:SubmitTask({ Name = "SendReward", Payload = { userId = 123 } })
roblox module 2025
005

RateLimiter

A limiter module with sliding window, token bucket, fixed window, and leaky bucket modes. I made it for the usual roblox problem of wanting decent protection without writing a different remote spam check every time. strict when it needs to be, still readable when you come back to it later.

local result = limiter:Check(player.UserId, "purchase") if not result.Allowed then return "slow down" end
roblox module 2025
006

StateMachine

A hierarchical state machine for game flow, npc logic, combat states, and other mode-heavy systems. Nested states, timed transitions, guards, and restore support are all there. it is mostly for when random flags and booleans stop being enough and the whole system starts stepping on itself.

machine:Send("AGGRO") print(machine:GetState()) machine:Send("KILLED")
roblox module 2025
007

CircuitBreaker

A circuit breaker wrapper for flaky operations. If something keeps failing, it stops hammering it, waits, probes recovery, and then closes again when it is safe. good around datastore calls, http calls, or anything else that likes dying at the worst time.

local result = cb:Execute("leaderboard", function() return DataStore:GetAsync("top10") end)
